Status: Reviewed

The report shows all PO in bucket for a particular Customer, bucket, customer_dc and date range.

Overview

The report shows the list of POs in a bucket for a particular passed Customer, Customer DC and date range if bucket is passed. If bucket is not passed this will also show the list of PO for a particular passed Customer, Customer DC and date range for pickslips of status id 1.

Algorithm

Get the list of all PO in bucket from ps, box and dem_pickslip tables.

Output

Item Summary Sort Sequence Detail
PO No 1 Purchase Order for given Customer.

Parameters

Name Default Detail
Customer Customer for which you want to display information.
Bucket Bucket for which you want to display information.
Customer Dist Center Customer Dist Center for which you want to display information.
From Date The date from which you want to display information.
To Date The date to which you want to display information.

Performance Detail

Expected Time for the report output is 1 sec

This Report displays the sub linear performance with respect to table ps, box and dem_pickslip. There is index based scan on the table ps and box. Index ps_customer_id_i and ps_bkt_fk_i used in the table ps, Index box_ps_fk_i used in the table box.

Behavior Change

  • Now if user want to see the PO between some date, he needs to pass the from and to date, report will display all the POs. The report will ignore the parameters which are left blank.

  • Now name of report is changed from Boxes In PO to PO LIST, because report is showing the list of POs based on passed parameters.

Issues

No Known Issue.

Parameter Validation

No special validations.

Query's documentation:

Query 1 will be executed when bucket is passed. Query 2 will be executed when bucket is not passed.

  • The query thats being used

    Query 1

       SELECT distinct (ps.po_id) AS PO_ID
  FROM ps ps, box box, po po
 WHERE ps.pickslip_id = box.pickslip_id
   AND ps.customer_id = po.customer_id
   AND ps.po_id = po.po_id
   AND ps.iteration = po.iteration
   AND (box.stop_process_date IS NULL OR
       (box.stop_process_reason NOT IN ('$PSCANCEL', '$BOXCANCEL')))
   AND ps.reporting_status is not null
   AND trunc(po.start_date) >= :FromDate
   AND trunc(po.start_date) <= :ToDate + 1
   AND ps.customer_dc_id = :Customer_dc
   AND ps.customer_id = :customer_id
   AND ps.bucket_id = :bucket_id

    Query 2

    SELECT DISTINCT (dps.customer_order_id) AS PO_ID
  FROM dem_pickslip dps
 WHERE dps.ps_status_id = '1'
   AND dps.upload_date IS NULL
   AND trunc(dps.delivery_date) >= :FromDate
   AND trunc(dps.delivery_date) <= :ToDate + 1
   AND dps.customer_dist_center_id = :Customer_dc
   AND dps.customer_id = :Customer_id
 ORDER BY PO_ID ASC
