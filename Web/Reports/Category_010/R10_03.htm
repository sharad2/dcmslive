This report displays detail of all unshipped order with the help of style, color and customer filters and also shows the count of POs by cancel date and import date.

Overview

This report displays detail of all unshipped order by style, color and customer and  how many pieces of SKU's for perticular PO for the passed filter as well as report  will  display the count of PO cancel date and import date wise.

We are displaying the three radio button 

  1. If you can check the first radio button  then report shows the detail PO wise that how may pieces of the SKU in the perticular  PO.
  2. If you can check the second radio button (i.e cancel date)  then report shows  the count of the PO per cancel date.
  3. If you check the last radio button then report shows the count of PO per importr date wise.

Algorithm

Information about the PO,pickslip,status within the specified is reterived from box table. We are reteriving the style, color,dimesion and sku_size information from the dem_pickslip_detail and retrieving name of the customer  from master_customer table.

Query documentation:

This report displays the detail of the PO 
      SELECT /*+rule*/
                PS.PICKSLIP_ID AS PICKSLIP_ID,
       PSDET.UPC_CODE AS UPC_CODE,
       MAX(PS.CUSTOMER_ORDER_ID) AS CUSTOMER_ORDER_ID,
       MAX(PSDET.STYLE) AS STYLE,
       MAX(PSDET.COLOR) AS COLOR,
       MAX(PSDET.SKU_SIZE) AS SKU_SIZE,
       MAX(PSDET.DIMENSION) AS DIMENSION,
       MAX(PS.CUSTOMER_ID) || ':  ' ||
       NVL(MAX(CD.NAME), 'Not in Customer Master') AS CUSTOMER_NAME,
       MAX(PS.DELIVERY_DATE) AS DELIVERY_DATE,
       MAX(PS.DC_CANCEL_DATE) AS DC_CANCEL_DATE,
       MAX(PS.PS_STATUS_ID) AS STATUS,
       SUM(PSDET.QUANTITY_ORDERED) AS TOTAL_PIECES,
       SUM(NVL(PSDET.QUANTITY_ALLOCATED, PSDET.QUANTITY_ORDERED)) -
       SUM(NVL(PSDET.QUANTITY_SHIPPED, 0)) AS SHORT_PIECES
  FROM DEM_PICKSLIP_DETAIL PSDET
 INNER JOIN DEM_PICKSLIP PS
    ON PSDET.PICKSLIP_ID = PS.PICKSLIP_ID
  LEFT OUTER JOIN MASTER_CUSTOMER CD
    ON PS.CUSTOMER_ID = CD.CUSTOMER_ID
 WHERE PS.PS_STATUS_ID IN (1, 99)
   AND PS.CUSTOMER_ID =:CUSTOMER_ID
   AND PSDET.STYLE = :STYLE
  and (:COLOR)
 GROUP BY PS.PICKSLIP_ID, PSDET.UPC_CODEid

This query shows the Count of POs import date wise

    SELECT /*+ RULE */
 PS.PICKSLIP_IMPORT_DATE AS IMPORT_DATE,
 SUM(PS.TOTAL_QUANTITY_ORDERED) AS ORDERED_QUANTITY,
 SUM(PS.TOTAL_DOLLARS_ORDERED) AS AMOUNT,
 TRUNC(SYSDATE) - TRUNC(PS.PICKSLIP_IMPORT_DATE) AS AGE,
 COUNT(DISTINCT PS.CUSTOMER_ORDER_ID) AS NO_OF_POS
  FROM DEM_PICKSLIP PS
 WHERE PS.PS_STATUS_ID IN (1, 99)
   AND PS.UPLOAD_DATE IS NULL
   AND ps.pickslip_import_date >= CAST(:Import_from_date as DATE)
   AND ps.pickslip_import_date < CAST(:Import_to_date as DATE) + 1
 GROUP BY PS.PICKSLIP_IMPORT_DATE

his query shows the Count of POs import date wise

    SELECT /*+ RULE */
 PS.CANCEL_DATE AS CANCEL_DATE,
 SUM(PSDET.QUANTITY_ORDERED) AS ORDERED_QUANTITY,
 SUM(PSDET.QUANTITY_SHIPPED) AS CHECKED_QUANTITY,
 SUM(PSDET.QUANTITY_ORDERED * EXTENDED_PRICE) AS AMOUNT,
 COUNT(DISTINCT PS.CUSTOMER_ORDER_ID) AS NO_OF_POS
  FROM DEM_PICKSLIP PS
  INNER JOIN DEM_PICKSLIP_DETAIL PSDET
  ON PS.PICKSLIP_ID = PSDET.PICKSLIP_ID
 WHERE PS.PS_STATUS_ID IN (1, 99)
  AND ps.cancel_date > = CAST(:from_date AS DATE)
 AND ps.cancel_date < CAST(:to_date AS DATE) +1 
 GROUP BY PS.CANCEL_DATE
