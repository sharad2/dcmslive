40.101 : Carton Aging Report

Author : Sanjeev Kumar

$Revision: 5601 $
Reviewed By: Manmohan Bisht
Tested By : Hemant K. Singh

For the specified period range, the report displays the cartons with their SKU details, which are lying in the specified area.

Overview

For the specific period (time period) and virtual warehouse, the report displays the label, season code, carton number, Area Location, receipt date and SKU details for a carton which are still lying in the specified area. the report displays the label, season code, carton number, location Id for the specified area, receipt date, style, color, size, dimension, and qty in broad categories:

  • Last 89 days
  • 90 to 179 days
  • 180 to 269 days
  • 270 to 359 days
  • 360 to 539 days
  • 540 to 719 days
  • 720+ days

Algorithm

For the particular time period, get label Id, location(If location is there in the area), season code, carton number, receipt date and SKU details of cartons which are still lying in the passed area from src_carton, src_carton_detail and master_style table.

Output

Item Summary Sort Sequence Detail
Carton No   Cartons lying in the BIR area.
Quality No   Quality code
Label No   Label Id assigned to the SKUs in the Carton.
Season Code No   Price Season Code for the cartons.
Receipt Date No 5 Date on which cartons were inserted.

Vwh No   Virtual Warehouse ID.

Parameters

Query's documentation:

  • The query gets the carton detail from src_carton and src_carton_detail as well as label Id from master_style table for the cartons which are lying in passed area for specified period of time. 
    SELECT CTN.carton_storage_area AS carton_storage_area,
       CTN.quality_code AS quality_code,
       CTN.carton_id         AS carton_id,
       CTN.price_season_code AS price_season_code,
       CTN.insert_date       AS receipt_date,
       DECODE(CTN.location_id,NULL,CTN.pallet_id,CTN.Location_Id)  AS carton_location,
       CTN.vwh_id            AS vwh_id,
       CTNDET.style          AS style,
       CTNDET.color          AS color,
       CTNDET.dimension      AS dimension,
       CTNDET.sku_size       AS sku_size,
       CTNDET.quantity       AS quantity,
       STY.label_id          AS label_id
FROM src_carton ctn
       LEFT OUTER JOIN src_carton_detail ctndet ON ctndet.carton_id = ctn.carton_id
       LEFT OUTER JOIN master_style sty ON sty.style = ctndet.style
 WHERE 1=1
   [$:carton_storage_area$AND ctn.carton_storage_area = :carton_storage_area$]
   [$:vwh_id$AND ctn.vwh_id = :vwh_id$]
   [$:EqualsTo != '8'$AND TRUNC((SYSDATE - ctn.insert_date) / 90) IN({0})$]
   [$:quality_code$AND ctn.quality_code = :quality_code$]
   [$:GreaterThan = '8' $AND TRUNC((SYSDATE - ctn.insert_date) / 90) >= :quarterly_year$]

Validation details

Bellow queries may have the where clause in the [] brackets. It means these are the optional parameters. If you do not want to pass these parameters just remove the [] section.
