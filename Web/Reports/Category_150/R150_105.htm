This report displays the productivity information for specified reworker for specified day and shift. It displays the rework codes along with the reworked cartons, units and pieces. This will help in deciding on the Rework Bonus for reworkers.

Overview

When the cartons are received in the warehouse, their contents are verified for quality and quantity. If there is some packaging error in the content, then the carton is marked yellow and sent to REW area (through K2P) along with the rework code(s), so that the packaging errors can be reworked. In the rework area, reworker works on cartons depending upon the rework code(s) assigned to them. This report displays the information regarding the rework undertaken by a specified reworker on a specified date and shift.

The report can be helpful

  • for monitoring the productivity of reworker.
  • for calculating Rework Bonus.
  • for identifying the star performers and the inefficient workers
 

Output

Item Summary Sort Sequence Detail
Rework Codes No 1 The type of Rework undertaken.
Cartons Sum The number of cartons on which the particular rework performed.
Units Sum Total number of units reworked.
Pieces Sum Total number of pieces reworked.

Parameters

Name Default Detail
Reworker None The Reworker for which you wish the see the information.
Process Date None The date on which the rework process was undertaken.
Shift All The Shift for which, you wish to view the rework information.

Performance Detail

Expected Time for the report output is 1 seconds

The query for this report will exhibit linear performance with respect to the number of records in carton_productivity table for specified date range. The query also implements a join with master_sku table. Since the records in master_sku are not likely in increase over a period of time, the performance will not depend upon the master_sku table.

Details

This report displays the records according to shift timings. For example, if a Reworker works on cartons on the 10-August-2006 at 11.59PM and works on another carton on the 11-August-2006 at 12.01AM. This means both the cartons were reworked with in same shift (second shift). The report will show both of these carton records in single row against 10-August-2006.

Query's documentation:

  • This query will get the rework information for specified reworker on a specified date and shift. All this information will be fetched from carton_productivity table. 
     WITH carton_details AS( 
 SELECT UPPER(cp.work_codes) AS work_codes,
        CASE
          WHEN (cp.process_start_date - TRUNC(cp.process_start_date)) * 24 <= 5.1 THEN
           TRUNC(cp.process_start_date - 1)
          ELSE
           TRUNC(cp.process_start_date)
        END AS shift_start_date,
        cp.carton_id AS carton_id,
        cp.carton_quantity AS carton_quantity,
        (cp.carton_quantity / ms.pieces_per_package) AS pieces_per_package
   FROM carton_productivity cp
   LEFT OUTER JOIN master_sku ms ON cp.upc_code = ms.upc_code
  WHERE cp.module_code = :module_code
    AND cp.work_type = :work_type
    AND cp.operator_id = :operator
    AND cp.work_codes IS NOT NULL
    AND cp.process_start_date >= (:process_start_date)
    AND cp.process_end_date < (:process_start_date) + 2
    AND CASE
  WHEN(cp.process_start_date - TRUNC(cp.process_start_date)) * 24 BETWEEN 5.1 AND 15.5 THEN 1 ELSE 2 END = :shift_number)
 SELECT cd.work_codes as work_codes,
        cd.shift_start_date as shift_start_date,
        count(cd.carton_id) as carton_id,
        sum(NVL(cd.carton_quantity, 0)) as carton_quantity,
        sum(NVL(cd.pieces_per_package, 0)) as pieces_per_package
   FROM carton_details cd
  WHERE cd.shift_start_date >= (:process_start_date)
    AND cd.shift_start_date < (:process_start_date) + 1
  GROUP BY work_codes, shift_start_date
  ORDER BY work_codes ASC
