This report shows the summary of each checker for a given date range and also on shift bases.

Overview

This report shows the productivity of each checker for a given date or date range and also on shift bases. The report is helpful in monitoring the productivity. The productivity is measured by number of boxes and pieces checked by a checker in a day.

Output

Item Summary Sort Sequence Detail
OPERATOR No 1  The Checker,who performed Checking.
Session ID No 2 The id of the session against which checker performed the operation
Warehouse Location No   Warehouse Location of Operator
Operation Date No    The date on which, checking operation was performed by the checker
Start Time No   The time on which work was started on the particular process by the checker
End Time No   The time on which work was ended on the particular process by the checker
Number of PDs Sum   Number of pickslips, processed by the checker on the particular date.
Number of Boxes Sum   The number of boxes, processed by the checker on the particular date.
Number of Pieces Sum   Number of pieces, processed by the checker on the particular date.
Number of UPC Scans Sum   Number of UPC scans, done by the checker on the particular date.
Number of COO Scans Sum   Number of Country of Origin scans, done by the checker on the particular date.
Number of Qty scans Sum   Number of qty mode scans, done by the checker on the particular date.
User Setup Date None    Date when user had been created.

Parameters

Name Default Detail
Operation Date Last 7 days period Date on which checking operation was performed by the checker.
Shift All Shifts Checker can work in any shift. Checker can perform the checking operation in multiple shifts. Specify a shift if you interested in a specific shift.
Warehouse Location All Warehouse Location where checker worked.

Behavior Change

  • Summary Check box is removed, instead we are recommending to use Collaps button it will show you summary for all the Checkers.

Query's documentation:

  • The query is the main query of the report specified operation date. 
    with Chk_Productivity AS (
SELECT bp.process_id,
       bp.ucc128_id,
       bp.operator,
       count(distinct bp.ucc128_id) OVER(PARTITION BY  bp.operator, bp.process_id) as count_cartons,
       count(distinct bp.pickslip_id) OVER(PARTITION BY  bp.operator, bp.process_id) as count_pickslips,
       min(bp.operation_start_date) OVER(PARTITION BY  bp.operator, bp.process_id) as start_time,
       max(bp.operation_end_date) OVER(PARTITION BY  bp.operator, bp.process_id) as end_time,
       max(bp.operation_start_date) OVER(PARTITION BY bp.operator, bp.process_id, bp.ucc128_id) as max_start_date,
       bp.num_of_pieces AS num_of_pieces,
       sum(bp.num_of_upc_scans) over(partition by bp.operator, bp.process_id) as sum_num_of_upc_scans,
       sum(bp.num_of_coo_scans) over(partition by bp.operator, bp.process_id) as sum_num_of_coo_scans,
       sum(bp.num_of_qty_mode_scans) over(partition by bp.operator, bp.process_id) as sum_num_of_qty_mode_scans,
       sum(bp.number_of_unscans) over(partition by bp.operator, bp.process_id) as sum_number_of_unscans,
       ROW_NUMBER() OVER(PARTITION BY bp.operator, bp.process_id, bp.ucc128_id ORDER BY bp.operation_start_date DESC) as row_sequence,
       max(au.created) OVER(PARTITION BY  bp.operator, bp.process_id) as user_setup_date   
  FROM box_productivity bp
  LEFT OUTER JOIN all_users au on au.username = bp.operator  
 WHERE bp.operation_code = '$CHECKING'
   AND bp.operation_start_date >= CAST(:operation_start_date_from AS DATE)
   AND bp.operation_start_date < CAST(:operation_start_date_to AS DATE) + 1
   
   AND bp.warehouse_location_id = :warehouse_location_id
)
SELECT cp.operator AS operator,
       cp.process_id AS process_id,      
       trunc(cp.max_start_date) AS max_start_date,
       MAX(cp.start_time) as start_time,
       MAX(cp.end_time) as end_time,
       NVL(MAX(cp.count_cartons),0) AS count_cartons,
       NVL(MAX(cp.count_pickslips),0) AS count_pickslips,
       NVL(SUM(cp.num_of_pieces),0) AS num_of_pieces,
       NVL(MAX(cp.sum_num_of_upc_scans),0) AS sum_num_of_upc_scans,
       NVL(MAX(cp.sum_num_of_coo_scans),0) AS sum_num_of_coo_scans,       
       NVL(MAX(cp.sum_num_of_qty_mode_scans),0) AS sum_num_of_qty_mode_scans, 
       NVL(MAX(cp.sum_number_of_unscans),0) AS sum_number_of_unscans,
       MAX(cp.user_setup_date) as user_setup_date  
  FROM Chk_Productivity cp
 WHERE cp.row_sequence = 1
 GROUP BY cp.operator, cp.process_id,trunc(cp.max_start_date)

Validation details

Bellow queries may have the where clause in the [] brackets. It means these are the optional parameters. If you do not want to pass these parameters just remove the [] section.

Query:

SELECT boxpro.operator AS "OPERATOR",
boxpro.process_id AS "PROCESS_ID",
TRUNC(boxpro.operation_start_date) AS "OPERATION_START_DATE",
min(boxpro.operation_start_date) AS "START_TIME",
max(boxpro.operation_end_date) AS "END_TIME",
COUNT(DISTINCT BOX.Pickslip_Id) AS "PDS",
COUNT(DISTINCT boxpro.ucc128_id) AS "BOXES"
FROM dcms8.box_productivity boxpro, dcms8.box box, dcms8.ps ps
WHERE boxpro.operation_code = '$CHECKING'
AND boxpro.ucc128_id = box.ucc128_id(+)
AND box.pickslip_id = ps.pickslip_id(+)
AND boxpro.process_id=::PROCESS_ID
GROUP BY boxpro.operator,
boxpro.process_id,
TRUNC(boxpro.operation_start_date)
