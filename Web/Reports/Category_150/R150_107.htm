This report displays the productivity information of the kitters for specified period of time and shift. This report can be drilled down on kitter field to see the carton details.

Overview

When the cartons are received in the warehouse, their contents are verified for quality and quantity. If there is some packaging error then the carton is marked yellow and rework codes are assigned as per the rework required. These cartons are scanned by K2P and then sent to be reworked, so that the packaging errors can be corrected. Kitter has option to un-scan a carton also. This un-scan are not captured as productivity information; just the scans are captured. This report displays the productivity of the kitters. This means, during a given period of time, how many cartons were scanned by the kitters.

The report can be helpful
  • For monitoring the productivity of kitters.
  • For identifying the star performers and the inefficient workers.

Algorithm

Get the kitter productivity information for specified time period from CARTON_PRODUCTIVITY table.

Output

Item Summary Default Sort Sequence Detail
Kitter No 1 Name of the kitter.
Process Date No 2 Date when kitting was performed.
Total Pallets Sum   Total number of pallets, whose cartons were scanned by the kitter.
Total Cartons Sum   Total number of cartons scanned by the kitter.
Total Pieces Sum   Total number of pieces of cartons scanned by the kitter.
User Setup Date None    Date on which the User or operator had been created.

Parameters

Name Default Detail
Process date range Last seven days The date range for which, you wish to view the kitting productivity information.
Shift All The shift for which, you wish to view the kitting productivity information. There are two shifts and each kitter can work in any shift or both the shifts. Specify a shift if you interested in a particular shift.

Performance Detail

Expected Time for the report output is 1 seconds

The query of this report will exhibit linear performance with respect to number of records in carton_productivity table qualified for a specified date range.

Details

This report displays the records according to shift timings.

Parameter Technical Details

  • Process Start Date (Type = Date): You can specify the date or date range to see the kitter productivity for this date or date range. This is a date range control.
  • Shift (Type = NUMBER): This is a drop down list parameter control. Specify a shift if you are interested in a particular shift. Default value text is "All".
    • Value = 1, Text = First
    • Value = 2, Text = Second

Query's documentation:

  • This query will get the K2P productivity details from table carton_productivity. 
     
  WITH kitter_prod AS         (         
SELECT c.operator_id AS operator_id,
       CASE
         WHEN (c.process_start_date - TRUNC(c.process_start_date)) * 24 <= 5.1 THEN
          TRUNC(c.process_start_date - 1)
         ELSE
          TRUNC(c.process_start_date)
       END AS shift_start_date,
       c.pallet_id AS pallet_id,
       c.carton_id AS carton_id,
       c.carton_quantity AS carton_quantity
  FROM carton_productivity c
 WHERE c.module_code = :module_code
   AND c.action_code = :action_code
   AND c.PROCESS_START_DATE >= (:process_start_date_from)
   AND c.PROCESS_START_DATE < (:process_start_date_to) + 2
   AND CASE
 WHEN(c.process_start_date - TRUNC(c.process_start_date)) * 24 BETWEEN 5.1 AND 15.5 THEN 1 ELSE 2 END = :shift_number)
SELECT kp.operator_id AS operator_id,
       NVL(COUNT(DISTINCT kp.pallet_id), 0) AS count_pallet_id,
       NVL(COUNT(kp.carton_id), 0) AS count_carton_id,
       NVL(SUM(kp.carton_quantity), 0) AS sum_carton_quantity,
       kp.shift_start_date as shift_start_date,
       max(au.created) as user_setup_date
  FROM kitter_prod kp
  LEFT OUTER JOIN all_users au on au.username = kp.operator_id
                              AND kp.shift_start_date >=
                                  (:process_start_date_from)
                              AND kp.shift_start_date <
                                  (:process_start_date_to) + 1
 GROUP BY kp.operator_id, shift_start_date
 ORDER BY operator_id ASC, shift_start_date ASC
