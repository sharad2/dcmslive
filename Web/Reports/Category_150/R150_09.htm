This report displays the productivity of V2P (VAS to pallet) for the specified period of time. VAS productivity is defined as the number of boxes scanned in the VAS to pallet process.

Overview

Report displays the information of the boxes scanned by the users in V2P for the given date range. Report also displays the user wise information of the first and the last box scanned in V2P on daily basis. This will help managers to see that who is performing well in the V2P process by checking the number of boxes done by the users per day. More boxes means more productivity. Hours are being displayed just to let the viewer know that how much time (in Hrs.) workers are spending daily in between the first and last box scanned in V2P. For dedicated V2P users this report will definitely help on giving incentives to good performers and finding out black sheep.

The report can be helpful

Algorithm

Get the box level VAS information like the first and last box scanned on from BOX_VAS table and VAS description from the tab_vas table.

Query's documentation:

  • This query will get VAS productiviy information for specified date range. 
    SELECT  trunc(A.DATE_CREATED) AS V2P_Date,
        A.CREATED_BY AS Operator,
        rtrim(SYS.STRAGG(UNIQUE B.description || ','),',') AS Applied_VAS,
        COUNT(DISTINCT A.UCC128_ID) AS Boxes_Scanned,
        MIN(A.DATE_CREATED) AS First_BOX_Scanned_on,
        MAX(A.DATE_CREATED) AS Last_BOX_Scanned_on,
        ROUND((MAX(A.DATE_CREATED) - MIN(A.DATE_CREATED)) * 24, 2) AS Hours,
        sum(bd.current_pieces) as total_pieces
   FROM BOX_VAS A
   left outer join boxdet bd
   on a.ucc128_id = bd.ucc128_id
   INNER JOIN tab_vas b
   ON a.BOX_PROCESS_CODE = b.vas_code
  WHERE 1=1 
     and a.DATE_CREATED > = CAST(:vas_date_from AS DATE)
     AND a.DATE_CREATED < = CAST(:vas_date_to AS DATE) +1
  GROUP BY TRUNC(A.DATE_CREATED), A.CREATED_BY
