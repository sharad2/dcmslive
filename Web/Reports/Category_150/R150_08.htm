$Revision: 520 $
Status : Reviewed

This report displays the productivity information of reworkers for specified period of time and shift. This will help in deciding the rework bonus for reworkers. This report can be drilled down on reworker field to see the carton details.

Overview

When the cartons are received in the warehouse, their contents are verified for quality and quantity. If there is some packaging error then the carton is marked yellow and rework codes are assigned as per the rework required. These cartons are scanned by K2P and then sent to be reworked, so that the packaging errors can be corrected. Reworker works on cartons depending upon the rework code assigned to the cartons. Also reworker has the option not to perform any rework on a carton for reasons like rework not required, components not available, etc. Such cartons are captured for productivity also. This report displays the productivity of the reworkers. This means, during a given period of time, how many cartons were reworked by the reworkers.

The report can be helpful

  • for monitoring the productivity of reworkers.
  • for monitoring the count of cartons not reworked.
  • for calculating rework bonus.
  • for identifying the star performers and the inefficient workers.

Algorithm

Get the productivity information for specified time period from CARTON_PRODUCTIVITY table. Read SKU Unit(one SKU unit can have two or more SKU pieces) information from MASTER_SKU table.

Query's documentation:

  • This query will get rework productiviy information for specified date range from carton_productivity table 
    WITH rework_prod as ( 
SELECT CP.OPERATOR_ID AS OPERATOR_ID,
       CASE
         WHEN (cp.process_start_date - TRUNC(cp.process_start_date)) * 24 <= 5.1 THEN
          TRUNC(cp.process_start_date - 1)
         ELSE
          TRUNC(cp.process_start_date)
       END AS shift_start_date,
       cp.process_start_date AS process_start_date,
       DECODE(CP.WORK_CODES, NULL, NULL, CP.CARTON_ID) AS CARTONS,
       DECODE(CP.WORK_CODES, NULL, CP.CARTON_ID) AS NO_REWORK_CARTONS,
       DECODE(CP.WORK_CODES, NULL, NULL, CP.CARTON_QUANTITY) AS PIECES,
       DECODE(CP.WORK_CODES,
              NULL,
              NULL,
              CP.CARTON_QUANTITY / MS.PIECES_PER_PACKAGE) AS UNITS
  FROM carton_productivity cp
  left outer join MASTER_SKU MS on cp.upc_code = MS.UPC_CODE
 WHERE cp.module_code = :module_code
   AND cp.work_type = 'REWORK'
   AND cp.process_start_date >= :process_start_date_from
   AND cp.process_start_date < :process_start_date_to + 2
   AND CASE
 WHEN(cp.process_start_date - TRUNC(cp.process_start_date)) * 24 BETWEEN 5.1 AND 15.5 THEN 1 ELSE 2 END = :shift_number)
SELECT rp.operator_id,
       count(rp.cartons) AS carton_count,
       count(rp.no_rework_cartons) AS no_rework_cartons_count,
       NVL(sum(rp.pieces), 0) AS carton_quantity,
       NVL(sum(rp.units), 0) AS units,
       rp.shift_start_date as shift_start_date,
       max(au.created) as user_setup_date
  FROM rework_prod rp
  LEFT OUTER JOIN all_users au on au.username = rp.operator_id
 WHERE rp.shift_start_date >= :process_start_date_from
   AND rp.shift_start_date < :process_start_date_to + 1
 Group BY rp.operator_id, rp.shift_start_date
 ORDER BY operator_id ASC, shift_start_date ASC
