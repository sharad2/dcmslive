150.104 : Kitter Productivity Report

Author : Manmohan Bisht

$Revision: 520 $
Status : Reviewed
Reviewed By : Pravin Awasthi

This report displays the productivity information of the kitter for specified date and shift. It displays the rework codes along with the scanned cartons, and pieces under these cartons.

Overview

When the cartons are received in the warehouse, their contents are verified for quality and quantity. If there is some packaging error in the content then the carton is marked yellow. Verifier will mark rework codes against this carton as per the rework required. Carton will be marked done and system will maintain carton in ‘MAT’ area. Now Kitter (K2P) will scan these yellow cartons from ‘MAT’ area and sort these to pallet manually. Kitter marks pallet closed after scanning a maximum of 28 cartons per pallet. However, kitter will not be forced to close a carton after putting 28 cartons on a pallet. Instead Kitter will be warned/informed in some manner that 28 cartons have been placed on the pallet and it will be confirmed from him/her if he/she wants to continue putting more cartons on pallet.

Kitter will have option to un-scan a carton before a pallet is closed. This un-scan will not be captured as productivity information; just the scan will be captured. K2P marks carton for ‘REW’ area, so that the packaging errors can be reworked. This report displays the information of cartons scanned by a specified Kitter on a specified date and shift.

The report can be helpful

  • For monitoring the productivity of kitters.
  • For identifying the star performers and the inefficient workers.

Algorithm

Get the kitter productivity information for specified Kitter and date and shift, from carton_productivity table.

Output

Parameters

Performance Detail

Expected Time for the report output is 1 seconds

The query for this report will exhibit linear performance with respect to the number of records in carton_productivity table for specified date range.

Parameter Technical Details

  • Kitter (Type = String): Specific Kitter, who performed the scan operation on the date and shift.
  • Process Date (Type = Date): Date on which scan operation was performed by the kitter.
  • Shift (Type = Number): This is a drop down list parameter control. Each Kitter can work in any shift. A Kitter can perform the scan operation in multiple shifts. Specify a shift if you interested in a specific shift. Default value text is "All".
    • Value = 1, Text = First
    • Value = 2, Text = Second

Query's documentation:

Bellow queries may have the where clause in the [] brackets. It means these are the optional parameters. If you do not want to pass these parameters just remove the [] section.

  • This query retrieves the carton details for the specified operator and date on which operation was performed. 
      WITH kitter_productivity AS   (         
SELECT UPPER(C.WORK_CODES) AS WORK_CODES,
       CASE
         WHEN (C.process_start_date - TRUNC(C.process_start_date)) * 24 <= 5.1 THEN
          TRUNC(C.process_start_date - 1)
         ELSE
          TRUNC(C.process_start_date)
       END AS shift_start_date,
       C.process_start_date,
       C.carton_id AS carton_id,
       C.carton_quantity AS carton_quantity
  FROM CARTON_PRODUCTIVITY C
 WHERE C.MODULE_CODE = :module_code
   AND C.ACTION_CODE = :action_code
   AND C.OPERATOR_ID = :operator_id
   AND C.process_start_date >= :process_start_date
   AND C.process_start_date < :process_start_date + 2
   AND CASE
 WHEN(C.process_start_date - TRUNC(C.process_start_date)) * 24 BETWEEN 5.1 AND 15.5 THEN 1 ELSE 2 END = :shift_number)
SELECT kp.WORK_CODES AS work_codes,
       count(kp.CARTON_ID) as carton_count,
       sum(kp.CARTON_QUANTITY) AS carton_quantity
  FROM kitter_productivity kp
 WHERE kp.shift_start_date >= :process_start_date
   AND kp.shift_start_date < :process_start_date + 1
 GROUP BY work_codes
 ORDER BY work_codes ASC
