This report shows those locations where SKU's are assigned and locations which are open for pitching i.e. unassigned locations which can be assigned and made available for pitching.

Overview

This report shows the details of all Locations in the specified warehouse, where the specified SKU is assigned i.e locations, from where pitching for the specifed SKU can be done. You can view the SKU Details of assigned location based on the following optional criterias:
  • Style
  • Color
  • Dimension
  • SKU Size
  • Label
If you wish to see the unassigned locations i.e location which are open for pitching (open for pitching means location can be assigned a SKU and consequently it will be a Pitch Location). You can select the unassigned tab and click Go button, in this case report will show only locations and thier capacity along with the Warehouse ID.
Note: Selecting unassigned only check box option will degrade the report performance.

Performance Detail

Expected Time for the report output is 1 second

The query will exhibit sub linear performance with repect to number of records in ialoc and ialoc content. The query also does a index based search on master_sku and master_style table. Since records in these tables are not likely to increase drastically, the query will display good performance. The query took 0.047 seconds for 18 qualified SKUs. When the query is run for unassigned location, it performs an index(IALOC_IA_FK_I) based search on ialoc table. This query will exhibit a logrithmic performance with respect to the number of records in ialoc table. Query took 0.047 seconds 650 qualified locations.

Details

Since we are using outer join while joining ialoc and ialoc_content table, those location can also be retrieved, which are unassigned. Therefore, on extra where clause (ialoc.assigned_upc_code is not null) have been implemented in the query.

Behavior Change

  • Provided two new columns 'Std. Case Qty' and 'Estimated Cartons'. 'Estimated Cartons' column will show the maximum number of cartons which can be repacked with pieces at location, considering standard case quantity.

Query Documentation

QUERY : Below is report Query. 
             select /*+index (sku sku_uk) index (ialoc_content IALOCCONT_PK)  */
       ialoc.location_id as location_id,
       max(ialoc.vwh_id) as vwh_id,
       max(sku.style) as style,
       max(sku.color) as color,
       max(sku.dimension) as dimension,
       max(sku.sku_size) as sku_size,
       max(sku.standard_case_qty) as standard_case_qty,
       max(trunc(ialoc_content.number_of_units / decode(sku.standard_case_qty, 0, null, sku.standard_case_qty))) as Estimated_Cartons,
       max(ialoc_content.number_of_units) as number_of_units,
       max(label_id) as label_id,
       max(ialoc.assigned_upc_code) as assigned_upc_code,
       max(ialoc.assigned_upc_max_pieces) as assigned_upc_max_pieces,
       max(round((sku.weight_per_dozen / 12), 4)) as weight_each,
       max(round((sku.volume_per_dozen / 12), 4)) as cube_each
  from ialoc
  LEFT outer join ialoc_content on ialoc_content.ia_id = ialoc.ia_id
                               and ialoc_content.location_id =
                                   ialoc.location_id
     AND IALOC_CONTENT.IACONTENT_ID = IALOC.ASSIGNED_UPC_CODE
  LEFT OUTER JOIN master_sku sku on ialoc.assigned_upc_code = sku.upc_code
  left outer join master_style on master_style.style = sku.style
  LEFT OUTER JOIN IA ia1 on ia1.ia_id = ialoc.ia_id
 where assigned_upc_code is not null
   AND ialoc.location_type = 'RAIL'
   AND ia1.picking_area_flag = 'Y'
   AND sku.style = :style
   AND sku.color = :color                
   AND sku.dimension = :dimension
   AND sku.sku_size = :sku_size  
   AND master_style.label_id = :label_id                                                    
   AND ialoc.vwh_id = :vwh_id
 group by ialoc.location_id

Validation Queries

QUERY: Below query is showing SKU, label and Standard Case Qty. 
SELECT MST.LABEL_ID,
       MS.STYLE,
       MS.COLOR,
       MS.DIMENSION,
       MS.SKU_SIZE,
       MS.UPC_CODE,
       MS.STANDARD_CASE_QTY
  FROM MASTER_SKU MS
 INNER JOIN MASTER_STYLE MST
    ON MS.STYLE = MST.STYLE
 WHERE MS.UPC_CODE = '014671137873';
QUERY: below query is showing  VWH, Location, UPC and Capicity
SELECT I.VWH_ID, I.LOCATION_ID, I.ASSIGNED_UPC_CODE, I.ASSIGNED_UPC_MAX_PIECES
  FROM IALOC I
 WHERE I.LOCATION_ID = '911BB14AB06';
QUERY: below query is showing quantity,Expected Carton, Weight and Cube 
SELECT IC.NUMBER_OF_UNITS AS "Quantity",
       trunc(ic.number_of_units / ms.standard_case_qty) as "Expected Cartons",
       ROUND((MS.WEIGHT_PER_DOZEN / 12), 4) AS "Weight",
       ROUND((MS.VOLUME_PER_DOZEN / 12), 4) AS "Cube"
  FROM IALOC_CONTENT IC
  LEFT OUTER JOIN MASTER_SKU MS
    ON IC.IACONTENT_ID = MS.UPC_CODE
 WHERE MS.UPC_CODE = '014671137873'
   AND IC.LOCATION_ID = '911BB14AB06'
