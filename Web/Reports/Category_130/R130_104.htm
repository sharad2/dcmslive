130.104 : Inventory Transaction Details

Status :

This report displays the SKU information for inventory transacted for particular reason code and module code. The report output can be further retrieved by specifying inserted date and also a date range which is 7 days from the specified date.

Overview

Whenever, a specific inventory transaction takes place in DCMS, it is recorded in database along with appropriate details. The report displays the details of inventory transacted for a specified reason code, module code, transaction date (or dates), user and warehouse id. The details includes style, color, dimension and sku_size along with the transacted quantity.

Algorithm

For a specified reason code, module code, transaction date and warehouse id get the details of SKUs from src_transaction and src_transaction_detail table. Retreving style, color, dimesion, size, insert_date, insert_by, reason_code and module_code form the table src_transaction. and retreving the Transaction Pieces from table src_transaction_detail.

Output

Parameters

create or replace synonym src_transaction_detail for dcms4.src_transaction_detail;

Parameter Technical Details

  • Specify date to summarize inventory activity : Specify a particular date for the inventory activity.
  • Last week transactions: Check this box, if you wish to view the last week inventory.
  • Reason Code: Reason code for inventory transaction.
  • Module Code: Module code for inventory transaction.
  • Virtual Warehouse: This is a list parameter control. By default, inventory of all Warehouse is shown for each area. If your focus is on the inventory in a particular Warehouse, Specify that Warehouse Id. Null value text is "All".

Parameter Validation

  • No special validations.

Query's documentation:

  • The query will get the details of SKUs which are transacted against the specified reason code and module code and in a particular warehouse. 
                  SELECT trans.reason_code    AS REASON_CODE,
              trans.module_code           AS MODULE_CODE,
              trans.inserted_by           AS INSERTED_BY,
              trans.insert_date           AS INSERT_DATE,
              trans.style                 AS STYLE,
              trans.color                 AS COLOR,
              trans.dimension             AS DIM,
              trans.sku_size              AS SKU_SIZE,
              trans.vwh_id                AS VWH_ID,
              transdet.transaction_pieces AS TRANSACTION_PCS
              FROM src_transaction trans
              LEFT OUTER JOIN src_transaction_detail transdet ON transdet.transaction_id = trans.transaction_id
              WHERE transdet.transaction_pieces !=0
              AND (trans.INSERT_DATE >= cast(:insert_date as date) AND trans.INSERT_DATE < cast(:insert_date as date) + 1)
              AND (trans.INSERT_DATE >= cast(:insert_date as date)- 7 AND trans.INSERT_DATE < cast(:insert_date as date) + 1)
              AND trans.reason_code = cast(:reason_code as varchar2(255))
              AND trans.MODULE_CODE = cast(:module_code as varchar2(255))
              AND trans.VWH_ID = cast(:vwh_id as varchar2(255))

Validation details

The query below can validate the output.

Query:
          SELECT trans.reason_code           AS REASON_CODE,
            trans.module_code           AS MODULE_CODE,
            trans.inserted_by           AS INSERTED_BY,
            trans.insert_date           AS INSERT_DATE,
            trans.style                 AS STYLE,
            trans.color                 AS COLOR,
            trans.dimension             AS DIM,
            trans.sku_size              AS SKU_SIZE,
            transdet.transaction_pieces AS TRANSACTION_PCS
            FROM src_transaction trans
	        LEFT OUTER JOIN src_transaction_detail transdet 
             ON transdet.transaction_id = trans.transaction_id
            WHERE trans.reason_code is not null
            AND trans.INSERT_DATE Between
            TO_DATE('02/14/2006', 'MM/DD/YYYY') - 7 and
            TO_DATE('02/14/2006', 'MM/DD/YYYY') + 1
            AND trans.reason_code = 'D005'
            AND trans.MODULE_CODE = 'REC'
            AND trans.VWH_ID = '15'
            AND trans.style = '08155'
            AND trans.color = 'BDB'
            AND trans.dimension = 'A'
            AND trans.sku_size = '34'
            AND transdet.transaction_pieces is not null
Description:
          Above query can be used to verify the report data. Change the values of parameters accordingly.
