Overview

This report helps to repack the carton from Examinig area as well as clear picture of the inventory that in this time how many pieces lying per SKU in this area. Now you can find out specific SKU with use SKU parameter(Style, color, dimension and SKU size) as well as you can see all SKU of specific label and VWH.

Algorithm

Get all the inventory details in examining area from master table master_raw_inventory.Retrieving UPC and pieces per package from master_sku and style, color, dimension, size , pieces and quality of the SKU from master_raw_inventory table.

Output

Item Summary Sort Sequence Detail
Building No 1 building of the SKU. Visible when report is used as drill down to report 30.08
VWh No 2  The virtual warehouse ID for the SKU.
Style No 3 Style of the SKU
Color No 4 Color of the SKU
Dim No 5Dimension of the SKU
Sizes No 6SKU Sizes of the SKU

Performance Detail

Expected Time for the report output is 0.375 seconds

 

Behavior Change

  1. SKU parameter will help to find out specific SKU pieces in the examinig area as well as you can see all SKU for specific label.
  2. Now you can pass multiple value separated by comma in the style, color, dimension and Size parameter.
  3. Day parameter has been removed.
  4. Now you can check quality of the SKU.
  5. Report name changed 'Carton in examining area' to 'Inventory in examining area'.

Parameter Technical Details

  • VWh (Datatype =String):         This is a string  type parameter and its report is using to  display SKU and pieces virtual warehouse wise.
  • Area (Datatype =String):         This is a string type parameter and its report is using to  display SKU and pieces Inventory Area wise.
  • Style (Datatype =String):         This is a string type  parameter. User can use this parameter for specific Style.
  • Color (Datatype =String):       This is a string type  parameter.User can use this parameter for specific Color.
  • Dim (Datatype =String):         This is a string type  parameter.User can use this parameter for specific Dimension.
  • Size (Datatype =String) :        This is a string type  parameter.User can use this parameter for specific SKU Size.
  • Label (Datatype =String):      This is a string type  parameter User can use this parameter for see all inventory of the  specific Label.
  • Building (Datatype =String): This is a string type  parameter and its report is using to  display SKU and pieces Warehouse location wise.

Parameter Validation

Query's documentation:

           WITH Q1 AS
            (
              SELECT /*+ INDEX(box box_ia_id_copy_i)*/
             ms.style AS STYLE,
             ms.color AS COLOR,
             ms.dimension AS DIMENSION,
             ms.sku_size AS SKU_SIZE,
             sum(bd.current_pieces) AS TOTAL_QUANTITY,
             null as quality,
             null as building
              FROM BOX box 
              LEFT OUTER JOIN boxdet bd ON box.ucc128_id = bd.ucc128_id  
                                        AND box.pickslip_id = bd.pickslip_id
              LEFT OUTER JOIN master_sku ms ON bd.upc_code = ms.upc_code
             WHERE 
               bd.current_pieces > 0
               AND box.ia_id is not null
               AND box.ia_id <> :CancelArea
               AND box.stop_process_date IS NULL
               AND box.ia_id = :area
               AND box.vwh_id = :vwh_id              
               AND box.ia_id_copy is not null
             GROUP BY /*+ INDEX(box box_ia_id_copy_i)*/ ms.style,
                      ms.color,
                      ms.dimension,
                      ms.sku_size
            UNION
            SELECT ms.style AS STYLE,
                   ms.color AS COLOR,
                   ms.dimension AS DIMENSION,
                   ms.sku_size AS SKU_SIZE,
                   sum(iac.number_of_units) AS TOTAL_QUANTITY,
                   null as quality,
                   null as building
              FROM ialoc ia 
              LEFT OUTER JOIN ialoc_content iac ON ia.ia_id = iac.ia_id 
                                                 AND ia.location_id = iac.location_id
              LEFT OUTER JOIN master_sku ms  ON iac.iacontent_id = ms.upc_code
             WHERE 1=1
             and iac.ia_id !=:scraparea
              and iac.iacontent_type_id = :SkuTypeStorageArea
              AND iac.ia_id = :area
              AND ia.vwh_id = :vwh_id             
             GROUP BY ms.style, ms.color, ms.dimension, ms.sku_size
            ),
            Q2 AS
            (
            SELECT rawinv.style AS STYLE,
                   rawinv.color AS COLOR,
                   rawinv.dimension AS DIMENSION,
                   rawinv.sku_size AS SKU_SIZE,
                   sum(rawinv.quantity) AS TOTAL_QUANTITY,
                   max(rawinv.quality_code) as quality,
                   nvl(tia.warehouse_location_id,'Unknown') as building
              FROM master_raw_inventory rawinv
              left outer join tab_inventory_area tia on
              rawinv.sku_storage_area=tia.inventory_storage_area
             WHERE 1=1
             AND rawinv.sku_storage_area = :area
             AND rawinv.vwh_id = :vwh_id
             and rawinv.quality_code=:quality_code
             AND TIA.WAREHOUSE_LOCATION_ID = :warehouse_location_id
             and  TIA.WAREHOUSE_LOCATION_ID is null 
             GROUP BY rawinv.style, rawinv.color, rawinv.dimension, rawinv.sku_size, nvl(tia.warehouse_location_id,'Unknown')
             having(sum(rawinv.quantity)) <> 0
            )
            select q1.STYLE,q1.COLOR,q1.DIMENSION,q1.SKU_SIZE,q1.TOTAL_QUANTITY,q1.quality,q1.building
              from Q1 q1
            UNION
            SELECT q2.STYLE,q2.COLOR,q2.DIMENSION,q2.SKU_SIZE,q2.TOTAL_QUANTITY,q2.quality,q2.building from Q2 q2

Validation details

Bellow queries may have the where clause in the [] brackets. It means these are the optional parameters. If you do not want to pass these parameters just remove the [] section.

Query:
                SELECT MS.STYLE AS "STYLE",
              MS.COLOR AS "COLOR",
              MS.DIMENSION AS "DIMENSION",
              MS.SKU_SIZE AS "SKU_SIZE",
              SUM(BD.CURRENT_PIECES) AS "TOTAL_QUANTITY"
              FROM MASTER_SKU MS 
              RIGHT OUTER JOIN BOXDET BD ON MS.upc_code = BD.UPC_CODE                                         
              RIGHT OUTER JOIN BOX BOX ON BOX.UCC128_ID = BD.UCC128_ID
                                       AND BOX.Pickslip_Id = BD.Pickslip_Id
              WHERE 
              BD.CURRENT_PIECES > 0
              AND BOX.IA_ID IS NOT NULL
              AND BOX.IA_ID <> 'CAN'
              AND BOX.IA_ID = 'RST'
              AND BOX.VWH_ID = '15'
              AND BOX.IA_ID_COPY IS NOT NULL
              GROUP BY MS.STYLE, MS.COLOR, MS.DIMENSION, MS.SKU_SIZE
Description:
            Run the above query to validate the inventory quantity in RST area.
