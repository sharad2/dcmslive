30.03 : Cartons In Examining Area

Author : Gyaneshwar Lal

$Revision: 3933 $
Status: Reviewed
Reviewed By : Hemant K. Varshney

This report displays the list of cartons with other details which have stayed in the examining area for at least specified number of days.

Overview

This report displays the list of cartons with other details which are in examining area for specified number of days.
It also displays the details about each cartons as below:

  • The packing instructions i.e. how many pieces should be packed per package and what are the packaging preferences.
  • The relevant inputs for packing dept. like ; sale type and price season code.
  • Productivity and efficiency of examining department.
  • Viewing details of a particular SKU in the specified Area.

    Algorithm

    Get all the carton details in examining area from master table src_bundle_inventory.

    Output

    Item Summary Default Sort Sequence Detail
    VWH No 2 The virtual warehouse ID for the carton.
    Carton No 3  The carton ID for which the carton details are shown in examining area.
    Style No   Style of the SKU.
    Color No   Color of the SKU.
    Dimension No   Dimension of the SKU.
    Sku Size No   Size of the SKU.
    Days No  1 Days which are calculated as (sysdate - modified_date).
    Pieces Sum   Quantity stored in carton
    Pieces Per Package Sum   Pieces stored in cartons per package.
    Packaging Preference No   The pakaging preference for cartons.
    Sale Type No   The sale type for packing department.
    Price Season Code No   Price season code for packing department

    Parameters

    Name Default Detail
    No. of days0 The value should be greater than or equal to '0' and less than or equal to '180' days.
    Virtual Warehouse All Virtual warehouse ID for the cartons can selected from drop-down list. Select "All" to view from all the virtual warehouses.

    Performance Details

    Expected Time for the report output is 1 second

    The query exhibits linear performance with respect to the number of records in src_bundle_inventory. There is full scan on table src_bundle_inventory.

    Use the below script to create the synonyms

    create or replace synonym src_bundle_inventory for dcms4.src_bundle_inventory;

    Following appsetting is required:

    <add key="ExaminingAreaCode" value="EXM"/>

    Parameter Technical Details>
  • No. of Days (Type = XmlInt): Enter the number of days from which you wish to see the carton details.
  • Virtual Warehouse (Type = ID): This is a drop-down list parameter control. By default, inventory of all Warehouse is shown for each area. If your focus is on the inventory in a particular Warehouse, Specify that Virtual Warehouse. Null value text is "All".
    • Query to fill list values = "select tab_virtual_warehouse.vwh_id, tab_virtual_warehouse.vwh_id from tab_virtual_warehouse"
  • Area: This value is defaulted in web.config file. This report intents to show records from examining area so it has to be defaulted to 'EXM'

    • Parameter Validation

    • Validation Condition : (($DAYS >= 0) and ($DAYS <= 180))
    • ErrorMessage : Not needed, as the parameter control of the page itself does not accepts erronous value as input.

    Query's documentation:

    • The query will get all the cartons details which are in specified area from table src_bundle_inventory. 
                  SELECT src.vwh_id AS "VWH_ID",
            src.carton_id AS "CARTON_ID",
            src.style AS "STYLE",
            src.color AS "COLOR",
            src.dimension AS "DIMENSION",
            src.sku_size AS "SKU_SIZE",
            TRUNC(sysdate - src.modified_date) AS "DAYS",
            (src.quantity) - NVL(src.quantity_removed, 0) AS "PIECES",
            src.pieces_per_package AS "PIECES_PER_PACKAGE",
            src.Packaging_preference AS "PACKAGING_PREFERENCE",
            src.sale_type_id AS "SALE_TYPE_ID",
            src.price_season_code AS "PRICE_SEASON_CODE"
            FROM src_bundle_inventory src
            WHERE src.sku_storage_Area = 'EXM'
            AND (src.quantity) - NVL(src.quantity_removed, 0) > 0
            AND (sysdate - src.modified_date) >= :DAYS

    Validation details

    Below query has the where clause, please comment them out if you don't want them to be applied on the query.

    Query:
                  
        SELECT a.vwh_id, a.carton_id, a.style, a.color, 
            a.dimension, a.sku_size, 
            trunc( sysdate - a.modified_date) AS days,
            (nvl(a.quantity,0) - nvl(a.quantity_removed,0)) AS pieces,
            a.pieces_per_package, a.packaging_preference, 
            a.sale_type_id, a.price_season_code
            FROM src_bundle_inventory a 
            WHERE 
            a.sku_storage_area='EXM' -- FOR EXAMINING AREA
            AND (nvl(a.quantity,0) - nvl(a.quantity_removed,0)) > 0 
            AND (sysdate - a.modified_date) > 7 -- Change to view for different number of days.
    Description:
    
            The query will get the carton details in examining area for the specified number of days.
