DcmsLive Installation

Objective

Create a website which will serve the pages of DcmsLive web. IIS will be configured to pass intranet user identity to the application. After installation DcmsLive will be configuired to specify the database connect strings. Optionally, you can restrict access to some reports from specific users.

System Requirements

Windows Server 2003 R2 running IIS6.

Prerequisites

  • IIS Preinstalled:

    Ensure that IIS 6 is installed on the server. This web server is used for serving all pages in DcmsLive.

  • .NET Framework 4.0:

    Make sure .Net Framework 4.0 should be installed on the server. You can download it from one of many Microsoft links.

  • ODP.NET 11.2. This is required for connecting to the Oracle Server.

Deploy DcmsLive

  • Extract supplied zip file:

    You should first extract the contents of the zip file into a temporary directory (c:\temp). After extraction succeeds you will see a C:\temp\DcmsLive directory which will contain all the necessary files. You should then move this DcmsLive directory to a convenient location (c:\Dcms\DcmsLive is recommended).

  • Create Virtual Directory

    Create a Virtual directory  named DcmsLive. Any other name can also be specified but virtual directory with that name should not already exist. Specify c:\Dcms\DcmsLive as the path for this virtual directory. Disable anonymous access and enable windows authentication for this virtual directory.

Verify Installation

The installation need to be verified for the proper installation. Specify the URL http:/localhost/DcmsLive in your browser on machine in which you have just installed DcmsLive. You should see a list of available reports.
