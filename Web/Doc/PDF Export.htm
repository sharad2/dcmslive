Export to PDF

Features

  1. Fetches and displays the data for the report against the passed parameters
  2. In case of report with paging exports the selected page i.e. the page selected from the table and if run staight away for passed parameters then exports first page
  3. The format of any report isn't altered and remains the same
  4. Hiding each an every component from the page to export which is of least use for PDF eg. None of the button is displayed in PDF, recently viewed reports link is not been displayed, About box is also not displayed nor the filters etc
  5. Each page displays Title at the header and date of generating the PDF at footer
  6. All the links of the page are exported as plain text i.e. clicking those links in the PDF will not have any effect this rule also applies to table headers
  7. Each page displays its page number to get the index of current page of all the available pages
  8. Attributes for PDF have been set which can be viewed from its file -> properties, the attributes set by us are
    • Title: Title of report
    • Created Date: Current date
    • Author Name: Logged in user
    • Subject: Report Header
    • Keyword: Report description

Steps to perform for PDF export

  1. Paste the supplied assembly *.dll to the bin folder
  2. Insert a button in the ButtonBar, click handler of this button will perform the action for PDF export
  3. PageBase class within App_Code folder will be responsible for communicating with the supplied assembly and calling its function for PDF export and performing all the work on PDF document, it will also be responsible for communicating with other class about whether to perform PDF export or not
  4. Since we would like to remove left side bar in MaterPage while pdfexport so the placeholder containing the side bar should be set to visible = "false" while exporting
  5. Removing side bar should remove the padding of repots grid so that it can adjust to free left space so the css class for same should be set to empty
  6. For removing Expand All and Collapse All buttons while exporting the GridView need to perform some work, these work are in classes GridViewEx.cs and GridViewExMasterRow.cs so the dll for EclipseLibrary also need to be renewed
  7. Any work if to be performed on ServerControl has to be performed in the click event of export button
  8. Change the Default style sheet present within App Themes folder since the selector for main-content has changed from id selector to class selector

Limitations

  1. Table header i.e. column header for each page does not repeat as in case of HTML
  2. Table with large data may halt export
